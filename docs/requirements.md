# 제품 요구사항 문서: NEWSTART Festival Webapp

## 1. 프로젝트 개요

본 프로젝트는 2026 하계 청소년 페스티벌 **NEWSTART 생명의 열쇠를 찾아라** 운영을 위한 웹앱이다.

행사는 삼육대학교 캠퍼스 전체를 활용한 팀 기반 캠퍼스 퀘스트로 진행된다. 참가팀은 NEWSTART 8개 테마의 미션을 수행하며 점수를 획득하고, 각 테마를 클리어할 때마다 생명의 열쇠 코드 조각을 얻는다. 8개 테마를 모두 클리어하면 최종 문구 **홍명기홀로 오라!**가 완성되며, 홍명기홀에서 최종 인증을 진행한다.

## 2. 행사 운영 정보

- 행사명: 2026 하계 청소년 페스티벌: NEWSTART 생명의 열쇠를 찾아라
- 대상: 청소년 약 300명
- 팀 구성: 팀당 10명 내외
- 예상 팀 수: 약 30팀
- 장소: 삼육대학교 캠퍼스 전체
- 최종 장소: 홍명기홀
- 운영 방식: 웹앱 기반 캠퍼스 퀘스트
- 일반 미션 접근: 공통 QR 접속 후 미션 코드 입력
- 별도 QR 사용: 히든 이스터에그 QR, 홍명기홀 최종 인증 QR

## 3. 핵심 사용자

### 참가팀

- 팀장 또는 대표자가 웹앱에 팀 코드로 로그인한다.
- 현장 미션 안내판에 적힌 미션 코드를 입력한다.
- 미션 안내를 확인하고 인증 자료를 제출한다.
- 현재 점수, 추첨권 수, 클리어한 테마, 생명의 열쇠 코드 조각을 확인한다.
- 8개 테마 클리어 후 홍명기홀 최종 인증을 진행한다.

### 스태프

- 스태프 미션 현장에서 팀의 성공 여부를 확인한다.
- 관리자 화면에서 특정 팀의 미션 성공/실패를 처리한다.
- 현장 혼잡도나 안전 상황을 운영본부에 전달한다.

### 운영본부/관리자

- 팀 목록, 점수, 제출 내역, 인증 상태를 관리한다.
- 사진/텍스트/스크린샷 제출을 검토한다.
- 스태프 미션 승인 및 점수 수정을 수행한다.
- 공지와 돌발 미션을 발송한다.
- 최종 점수와 추첨권을 확정한다.

## 4. 핵심 운영 규칙

### 테마 클리어

- NEWSTART는 8개 테마로 구성된다.
- 각 테마에는 2개의 미션이 있다.
- 한 테마에서 1개 이상의 미션을 성공하면 해당 테마가 클리어된다.
- 한 테마의 2개 미션을 모두 성공하면 두 미션의 점수를 모두 획득한다.

### NEWSTART 완주

- 8개 테마를 모두 클리어하면 NEWSTART 완주로 인정한다.
- NEWSTART 완주 시 100점 보너스를 부여한다.
- 16개 전체 미션을 모두 완료하면 200점 올클리어 보너스를 부여한다.

### 최종 인증

- 8개 테마를 모두 클리어한 팀만 최종 인증을 성공 처리할 수 있다.
- 최종 인증 장소는 홍명기홀이다.
- 홍명기홀 최종 QR 인증 완료 시 추첨권 2장을 추가한다.
- 최종 인증은 팀당 1회만 인정한다.

## 5. 점수 체계

| 미션 유형 | 점수 | 기준 |
|---|---:|---|
| 쉬운 미션 | 30점 | QR 퀴즈, 간단 입력형 |
| 일반 미션 | 50점 | 사진 인증형, 탐색형, 이동형 |
| 도전 미션 | 80점 | 스태프 필요, 현장 수행형, 게임형 |

### 보너스 점수

| 조건 | 보너스 |
|---|---:|
| 8개 테마 모두 클리어 | +100점 |
| 16개 전체 미션 모두 완료 | +200점 |

## 6. 추첨권 계산

최종 상품 추첨권은 점수 구간에 따라 차등 지급한다.

| 최종 점수 | 추첨권 |
|---|---:|
| 300점 이상 | 1장 |
| 500점 이상 | 2장 |
| 700점 이상 | 3장 |
| 900점 이상 | 4장 |
| 홍명기홀 최종 인증 완료 | 추가 2장 |

- 한 팀이 받을 수 있는 추첨권은 최대 6장이다.
- 추첨권 계산은 웹앱에서 자동 산정한다.
- 관리자는 필요 시 수동 보정할 수 있어야 한다.

## 8. 웹앱 기능 요구사항

### 참가자 기능

1. 팀 코드 기반 로그인
2. 대시보드 표시
   - 팀명
   - 현재 점수
   - 현재 추첨권 수
   - 클리어한 테마 수
   - 생명의 열쇠 코드 현황
   - 공지사항
3. 미션 코드 입력
4. 미션 상세 확인
5. 미션 인증 제출
   - 퀴즈 정답 입력
   - 텍스트/카드 답안 입력
   - 사진 업로드
   - 스크린샷 업로드
   - 영상 대신 텍스트 또는 외부 제출 안내 가능
   - 스태프 승인 요청 상태 표시
6. 테마 클리어 및 코드 조각 표시
7. 최종 인증 화면
8. 추첨권 화면

### 관리자 기능

1. 관리자 로그인
2. 팀 목록 조회
3. 팀별 점수/추첨권/테마 클리어 현황 확인
4. 제출 내역 조회
5. 사진/텍스트/스크린샷 제출 승인 또는 취소
6. 스태프 미션 성공/실패 처리
7. 이스터에그 점수 지급
8. 점수 수동 수정
9. 공지 발송
10. 돌발 미션 생성 및 성공 처리
11. 최종 인증 여부 확인
12. 감사 로그 확인

## 10. 공통 QR + 미션 코드 방식

일반 미션마다 QR을 만들지 않는다.

- 공통 웹앱 접속 QR: 1개
- 참가자는 공통 QR로 웹앱에 접속한다.
- 각 미션 장소에는 미션 안내판과 미션 코드가 있다.
- 참가자는 웹앱에서 미션 코드를 입력한다.
- 해당 미션 상세 페이지가 열린다.

별도 QR은 다음에만 사용한다.

- 히든 이스터에그 QR
- 홍명기홀 최종 인증 QR

## 13. 안정성 요구사항

1. 이메일/문자 인증을 사용하지 않는다.
2. 팀 코드 기반 로그인으로 구현한다.
3. 모든 점수 부여는 중복 지급을 방지해야 한다.
4. 같은 팀이 같은 미션을 여러 번 제출해도 점수가 중복 반영되지 않아야 한다.
5. 관리자는 제출을 취소하거나 점수를 수동 수정할 수 있어야 한다.
6. 주요 액션은 audit log에 남겨야 한다.
7. 사진 업로드는 클라이언트에서 압축 후 업로드한다.
8. 영상 업로드는 기본 지원하지 않거나, 별도 제출 안내로 분리한다.
9. 실시간 랭킹은 Realtime 의존을 피하고 15~30초 polling으로 구현한다.
10. 운영 당일 백업 플랜을 고려해 CSV export 또는 관리자 수동 처리 기능을 제공한다.
